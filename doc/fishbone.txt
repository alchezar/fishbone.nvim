1. INTRODUCTION                                       *fishbone-introduction*

fishbone.nvim renders the whole file across the bottom row of the screen.
Every buffer line maps to a column on the bar - think of it as a one-line
minimap embedded in the global statusline.

It overlays multiple data sources on each cell: cursor position, viewport,
search matches, marks, diagnostics and git hunks. Clicking or dragging on
the bar moves the cursor.

==============================================================================
2. SETUP                                                     *fishbone-setup*

                                                          *fishbone.setup()*
require('fishbone').setup({opts})

  Initialises highlights, installs the statusline expression and sets
  `laststatus=3`. Must be called once at startup.

  Example: >lua
    require('fishbone').setup({
      colors = { cursor = '#FFFFFF' },
    })
<

3. CONFIGURATION                                            *fishbone-config*

`opts.colors`                                       *fishbone-config-colors*

  Each color is resolved in this order:
    1. Explicit value from `opts.colors.<name>`
    2. `fg` of a linked highlight group (see table below)
    3. Hardcoded fallback listed here

  Hex strings, used only as fallbacks when both explicit overrides and
  themed highlight groups are absent:

    cursor      '#FFFFFF'   current line marker
    search      '#FF77AA'   search-pattern hits
    mark        '#FFD866'   vim a-z marks / marks.nvim bookmarks
    viewport    '#888888'   lines visible on screen
    error       '#FC6161'   error diagnostics    | DiagnosticError
    warn        '#FFA348'   warning diagnostics  | DiagnosticWarn
    info        '#67D4F0'   info diagnostics     | DiagnosticInfo
    hint        '#C792EA'   hint diagnostics     | DiagnosticHint
    git_add         '#7FCC7F'   git added lines      | GitSignsAdd
    git_change      '#7FAFFF'   git changed lines    | GitSignsChange
    git_delete      '#FC6161'   git deleted marker   | GitSignsDelete
    git_add_dim     auto-blend  staged add           | GitSignsStagedAdd
    git_change_dim  auto-blend  staged change        | GitSignsStagedChange
    git_delete_dim  auto-blend  staged delete        | GitSignsStagedDelete
    base            '#444444'   empty-cell dots
    file        '#AAAAAA'   left-side file path
    dim         '#444444'   zero-count diag numbers
    info_txt    '#BBBBBB'   right-side L:C / P%

  The right column lists the highlight group fishbone reads when no
  explicit override is given. Highlights refresh on `ColorScheme`, so
  swapping themes automatically updates the bar.

4. MARKERS                                                 *fishbone-markers*

Each cell composes a top half (overview) and a bottom half (signals).

Top half - `▀` (highest wins):
  cursor    current cursor line
  search    lines matching the active /pattern
  mark      vim a-z marks, marks.nvim bookmarks
  viewport  lines currently visible on screen

Bottom half - `▄` (highest wins):
  error     warn      git_change   git_add    info    hint

Cells with both halves use `▀` with fg=top, bg=bottom. Cursor alone uses
`█`. Empty cells use `·`.

Deletes are an overlay. A column anchored to a removed git hunk gets `▁`
red on empty cells, or a red underline under the cell's existing glyph on
non-empty ones - so a delete next to a cursor or diagnostic doesn't hide
it. Distinct from the `▄` red used for error diagnostics.

Staged hunks (after `:Gitsigns stage_hunk`) keep showing on the bar in a
dimmer shade of the same color: dim green / blue / red for add / change /
delete. If a bright (unstaged) and a dim (staged) marker land on the same
column, the bright one wins. The dim color follows `GitSignsStagedAdd` /
`GitSignsStagedChange` / `GitSignsStagedDelete` when present, otherwise it
is blended automatically from the bright color toward `base`.

==============================================================================
5. MOUSE                                                     *fishbone-mouse*

Click on the bar to jump the cursor to that line. A jumplist entry is
recorded, so |CTRL-O| returns to the previous position.

Drag (hold left mouse, move) to scrub the cursor smoothly. Once a drag
starts on the bar, mouse Y can leave the statusline row - the cursor
still follows the X position; past the bar edges, X sticks at the extreme.

Works in terminal Neovim and Neovide. In Neovide, the plugin installs a
`<LeftMouse>` keymap because Neovide does not route clicks to statusline
`%@...@` regions.

==============================================================================
6. HIGHLIGHTS                                           *fishbone-highlights*

Generated by `setup()` from `opts.colors`. Override per-colorscheme if
needed:

  FbnT_<name>           top-only cells
  FbnB_<name>           bottom-only cells (incl. `_dim` for staged hunks)
  FbnT_<top>_B_<bot>    cells with both halves
  *_D                   same, with bright (unstaged) delete underline
  *_DD                  same, with dim (staged) delete underline
  FbnCursorBlock        cursor alone (`█`)
  FbnBase               empty-cell glyph (`·`)
  FbnDel                empty-cell delete marker (`▁`)
  FbnDelDim             empty-cell staged-delete marker (`▁`)
  FbnFile               left-side path
  FbnDim                zero-count diag numbers
  FbnInfoTxt            right-side L:C / P%
  FbnErrorTxt           error counter
  FbnWarnTxt            warn counter
